# AIQAHub 产品架构设计

## 1. 产品定位

**AIQAHub** 是一个 AI 驱动的智能质量保障平台，旨在为软件工程团队提供统一的测试管理、执行监控、报告分析和质量门禁能力。

### 1.1 核心价值主张

- **统一质量管控平面**：将分散的测试工具、执行环境、报告数据聚合到单一平台
- **AI 增强分析**：利用 AI 自动分析测试失败根因、提供修复建议、预测质量风险
- **灵活可扩展架构**：模块化单体设计，支持逐步演进为微服务架构
- **多维度治理可见性**：完整的审计追踪、事件流监控和质量指标看板

### 1.2 目标用户

| 用户角色 | 核心需求 | 使用场景 |
|---------|---------|---------|
| **测试工程师** | 测试用例管理、执行调度、结果分析 | 日常测试执行、失败分析、报告生成 |
| **开发工程师** | 质量门禁反馈、失败详情查看 | 提交代码后门禁检查、本地复现问题 |
| **QA 负责人** | 质量趋势分析、团队效能度量 | 周会质量汇报、风险识别、资源调配 |
| **DevOps 工程师** | 执行环境管理、连接器配置、通知策略 | CI/CD 集成、环境维护、告警配置 |
| **技术负责人** | 整体质量治理、合规审计 | 质量复盘、流程优化、合规检查 |

## 3. 核心功能模块详解

### 3.1 项目管理 (Projects)

**核心能力：**
- 项目元数据管理（名称、描述、环境、标签）
- 项目级配置隔离
- 项目成员与权限管理
- 项目级通知策略覆盖

**关键实体：**
- Project: 项目主体
- Environment: 环境（dev/staging/prod 等）
- ProjectMember: 项目成员关系

### 3.2 测试套件 (Suites)

**核心能力：**
- 测试用例/套件的注册与分类
- 套件版本管理
- 标签与元数据标注
- 套件与项目的关联

**关键实体：**
- Suite: 测试套件
- SuiteCase: 测试用例（可选扩展）
- SuiteVersion: 套件版本（可选扩展）

### 3.3 执行编排 (Executions)

**核心能力：**
- 执行触发（手动/API/定时/Webhook）
- 执行状态机流转
- 超时与重试策略
- 执行任务分发与并行
- 执行 artifact 收集

**关键实体：**
- Execution: 执行记录
- ExecutionTask: 执行子任务
- Artifact: 产物文件
- ArtifactLink: 产物关联

**执行状态机：**
```
PENDING → QUEUED → RUNNING → COMPLETED
                      ↓
                   FAILED / TIMEOUT / CANCELLED
```

### 3.4 报告聚合 (Reports)

**核心能力：**
- 多格式报告解析（JUnit XML、Playwright JSON、Allure 等）
- 报告数据归一化与聚合
- 测试结果趋势分析
- 失败分类与统计

**关键实体：**
- Report: 报告主体
- ReportCase: 测试用例结果
- ReportSuite: 测试套件结果

### 3.5 质量门禁 (Gates)

**核心能力：**
- 门禁规则定义（通过率、失败数、覆盖率等）
- 规则版本管理与审计
- 门禁评估与结果判定
- 门禁事件触发通知

**关键实体：**
- QualityRule: 质量规则
- QualityRuleRevision: 规则修订历史
- GateEvaluation: 门禁评估记录

**规则类型示例：**
- `pass_rate >= 95%`
- `failure_count <= 5`
- `flaky_rate <= 2%`
- `coverage_line >= 80%`

### 3.6 AI 分析 (AI)

**核心能力：**
- 失败根因自动分析
- 修复建议生成
- 类似失败案例关联
- 质量风险预测
- 分析历史追溯

**关键实体：**
- AIInsight: AI 洞察记录
- AIReplay: 分析回放对比

**分析场景：**
- 单次执行失败分析
- 多次执行趋势分析
- 项目整体质量健康度评估

### 3.7 资产库 (Assets)

**核心能力：**
- 测试资产版本管理
- 资产关联关系（测试数据、配置文件等）
- 资产变更追踪
- 资产引用溯源

**关键实体：**
- Asset: 资产主体
- AssetRevision: 资产版本
- AssetLink: 资产关联

### 3.8 通知中心 (Notifications)

**核心能力：**
- 多通道支持（邮件、钉钉、企业微信、Slack）
- 策略化路由（按事件类型、项目、级别）
- 发送失败回退机制
- 通知模板管理
- 发送历史与审计

**关键配置：**
- `notification_default_channel`: 默认通道
- `notification_policies`: 策略数组（JSON）
  - `scope_type`: global / project
  - `scope_id`: 作用域 ID
  - `event_type`: 事件类型
  - `channels`: 目标通道
  - `filters`: 过滤条件

**事件类型：**
- `execution.failed`
- `execution.timeout`
- `gate.failed`
- `gate.warning`
- `ai.analysis_ready`

### 3.9 治理中心 (Governance)

**核心能力：**
- 平台运营概览仪表盘
- 事件流实时监控
- 通知投递追踪
- 审计日志查询
- 配置变更历史
- 连接器健康状态

**监控分区：**
- Overview: 关键指标总览
- Event Stream: 事件时序流
- Notification Events: 通知投递详情
- Audit Log: 操作审计
- Connectors: 连接器状态

## 5. 产品成功指标

### 5.1 业务指标
- **平台采纳率**: 活跃项目数 / 总项目数
- **执行覆盖率**: 接入平台的执行数 / 总执行数
- **用户活跃度**: 日活/周活/月活用户数
- **通知到达率**: 成功发送通知数 / 总通知数

### 5.2 质量指标
- **平均修复时间 (MTTR)**: 从失败发现到修复的平均耗时
- **失败重现率**: 可重现失败数 / 总失败数
- **AI 分析准确率**: 用户采纳的 AI 建议数 / 总 AI 建议数
- **门禁拦截率**: 成功拦截的坏提交数 / 总坏提交数

### 5.3 效能指标
- **执行平均耗时**: 端到端执行时长趋势
- **环境准备时间**: 从触发到实际运行的耗时
- **报告生成延迟**: 执行完成到报告可用的耗时
