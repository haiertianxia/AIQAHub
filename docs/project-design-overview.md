## 项目简介

**AIQAHub** 是一个 AI 驱动的智能质量保障平台，旨在为软件工程团队提供统一的测试管理、执行监控、报告分析和质量门禁能力。

### 核心价值主张

1. **统一质量管控平面** - 将分散的测试工具、执行环境、报告数据聚合到单一平台
2. **AI 增强分析** - 利用 AI 自动分析测试失败根因、提供修复建议、预测质量风险
3. **灵活可扩展架构** - 模块化单体设计，支持逐步演进为微服务架构
4. **多维度治理可见性** - 完整的审计追踪、事件流监控和质量指标看板

### 后端分层架构

```
API 层 (app/api/v1/routes/)
    ↓
服务层 (app/services/)
    ↓
CRUD 层 (app/crud/)
    ↓
模型层 (app/models/)
    ↓
数据库 (SQLite/PostgreSQL)
```

### 核心技术实现

1. **执行状态机** - PENDING → QUEUED → RUNNING → COMPLETED/FAILED/TIMEOUT
2. **连接器框架** - 可扩展的测试工具集成 (Jenkins/Playwright 等)
3. **通知策略路由** - 按项目/事件类型路由到邮件/钉钉/企业微信
4. **治理事件投影** - 审计日志作为单一事实源，按需生成视图
5. **Celery 异步任务** - 报告解析、AI 分析等异步处理

## 核心决策记录

### ADR-001: 短期不支持多环境
**日期**: 2026-04-16
**决策**: Phase 1 只支持单一环境，多环境支持移至 Phase 3
**原因**:
- 聚焦核心价值，减少初期复杂度
- 快速交付可用版本，验证产品市场匹配
- 单一环境已满足大多数团队初期需求
**影响**:
- 项目与环境解耦推迟
- 环境级配置隔离推迟
- Phase 1 周期从 6-8 周缩短至 4-6 周

### ADR-002: 模块化单体架构起步
**决策**: 采用模块化单体架构，后续逐步演进为微服务
**原因**: 降低初期部署复杂度，便于快速迭代
**演进路径**: 模块化单体 → 事件驱动 → 核心服务拆分 → 微服务

### ADR-003: SQLite 开发，PostgreSQL 生产
**决策**: 开发环境使用 SQLite，生产环境使用 PostgreSQL
**原因**: 简化本地开发，生产环境保证性能和可靠性
